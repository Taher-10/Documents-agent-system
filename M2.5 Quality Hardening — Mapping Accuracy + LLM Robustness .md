### M2.5 Quality Hardening — Mapping Accuracy + LLM Robustness

### Summary
Harden the current `graph_v2` intelligence layer before API linking by implementing the 4 agreed fixes: stricter non-mappable filtering, bounded retry/backoff for Groq failures, prompt-size control for long sections, and a gold-set quality gate for the target PDF (`PQ-PROD-01 02 Procédure de gestion des gabarits.pdf`).

### Implementation Changes
1. **Strict rule-based non-mappable classifier in `graph_v2/nodes/react_mapper.py`**
- Keep current title blacklist and add deterministic type-based guard:
  - Default `NOT_APPLICABLE` for `metadata`, `definitions`, `references`, `process_diagram`.
  - Exception: allow mapping only if section text contains at least 1 modal term for selected language **and** cleaned text length `>= 240`.
- Add helper `_has_modal_commitment(text, language)` and reuse it for `has_commitments`.
- Keep current short-text guard (`<120`) for all section types.

2. **Prompt-size control + candidate reduction (same file)**
- Add `_compact_section_text(...)`:
  - Normalize whitespace.
  - If text length `<= 3500`, keep full text.
  - If `> 3500`, keep: first `1800` chars + last `1200` chars + up to `8` lines containing modal terms/keywords (`audit`, `nonconform`, `validation`, `traçabilité`, `client`).
- Add `_select_mapping_menu(...)`:
  - Build candidate list from `state["clause_menu"]`.
  - Score by title token overlap with compacted section text.
  - Keep top `30` clauses for mapping prompt.
- Keep assessment clause cap `8` (already in place).

3. **Groq retry/backoff + explicit failure taxonomy**
- Add `_invoke_structured_with_retry(llm, schema, prompt, stage, section_id)`:
  - Retries: `max_attempts=3` (initial + 2 retries).
  - Backoff: `1.0s`, `2.0s` with jitter `0-0.25s`.
  - Retry on transient/network/rate-limit/tool-use parse failures (including Groq `429`, `tool_use_failed`).
- Replace raw `.invoke(...)` calls with retry helper for both mapping and assessment.
- Replace generic `llm_parse_error` with specific gap codes:
  - `llm_mapping_error`
  - `llm_assessment_error`
  - `llm_rate_limited`
- Keep status `MISSING` for hard failures, but preserve diagnosability via gap code.

4. **Advice quality post-validation**
- Add `_normalize_advice(advice, clause_number)`:
  - Reject empty/whitespace or generic placeholders (`"N/A"`, `"incomplete documentation"`, `"improve documentation"`).
  - Replace with clause-specific fallback:  
  `f"Add documented controls and objective evidence for clause {clause_number} in this section."`
- Apply normalization before building final `SectionMatch`.

5. **Quality evaluation harness for this PDF**
- Add gold mapping file:
  - `agent_compliance/tests/fixtures/pq_prod_01_02_gold_mapping.json`
  - Contains per-section expected clause numbers (or accepted families) and `mappable` flag.
- Add evaluator script:
  - `agent_compliance/graph_v2/eval_mapping_quality.py`
  - Inputs: `doc_id`, `company_id`, `language`, `db_path`, `gold_path`.
  - Outputs: per-section comparison + micro precision/recall/F1 + NOT_APPLICABLE accuracy.
- Hard quality gate in script:
  - Precision `>= 0.45`
  - Recall `>= 0.60`
  - F1 `>= 0.50`
  - `NOT_APPLICABLE` correctness `>= 0.80`
- Exit code non-zero on gate failure.

6. **Progress documentation update**
- Update `agent_compliance/retrieval/08_progress.md`:
  - Add “M2.5 Quality Hardening” subsection.
  - Record commands/results for evaluator and updated tests.
  - Keep next phase as M2.6 API/report integration.
